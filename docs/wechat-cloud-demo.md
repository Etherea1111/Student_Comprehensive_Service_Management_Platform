# 微信云开发演示版部署说明

本项目已加入轻量云开发适配，用于课程/答辩体验版登录和核心功能演示。该方案不依赖自建 HTTPS 域名，也不直接访问阿里云后端。

## 适用场景

- 需要把小程序上传为体验版，让同学、老师扫码访问。
- 暂时没有已备案域名或 HTTPS 后端域名。
- 只需要演示登录、首页、知识库、党团流程、通知、审批、后台维护等核心路径。

## 本地配置

1. 使用微信开发者工具打开项目根目录。
2. 在工具顶部打开“云开发”，创建一个云开发环境。
3. 打开 `miniprogram/config/env.js`：
   - `useCloud: true` 保持开启。
   - 如工具不能自动识别默认环境，把云开发环境 ID 填到 `cloudEnvId`。
   - `cloudFunctionName` 保持 `api`。
4. 确认 `project.config.json` 中存在：
   - `miniprogramRoot: "miniprogram/"`
   - `cloudfunctionRoot: "cloudfunctions/"`

## 上传云函数

1. 在微信开发者工具左侧找到 `cloudfunctions/api`。
2. 右键选择“上传并部署：云端安装依赖”。
3. 部署完成后重新编译小程序。
4. 使用账号 `2024000001` 和任意非空密码登录；建议演示时仍使用 `123456`。

## 发布体验版

1. 云函数部署成功后，点击微信开发者工具“上传”。
2. 到微信公众平台的小程序后台选择该版本为体验版。
3. 添加体验成员后，成员即可扫码访问。

## 切回自建后端

如果后续绑定了 HTTPS 合法域名并继续使用阿里云后端：

1. 将 `miniprogram/config/env.js` 中的 `useCloud` 改为 `false`。
2. 将 `apiBaseUrl` 改为正式 HTTPS 地址，例如 `https://你的域名/api`。
3. 在微信公众平台配置 request 合法域名。

## 当前限制

- 云函数内置的是演示数据，不会写入 PostgreSQL。
- 文件上传只返回演示元数据，不保存真实附件。
- 若要做正式生产版，仍建议使用 HTTPS 后端 + PostgreSQL，并补全真实鉴权、审计和文件存储。
