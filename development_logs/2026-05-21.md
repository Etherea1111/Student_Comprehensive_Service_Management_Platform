# 2026-05-21 开发日志

## 今日目标

结合需求文档和要求明细中梳理出的 P0 级缺口，补齐小程序当前缺少的核心管理闭环，重点完善模块一、模块二的高完成度实现，并落地角色权限、敏感数据脱敏和管理员操作能力。

## 完成内容

1. 完善知识库维护闭环
   - 后端新增知识库管理列表接口。
   - 支持知识库条目编辑后重新进入待复核状态。
   - 支持老师或具备复核权限的管理员发布知识库草稿。
   - 支持退回待复核草稿。
   - 支持归档已发布知识库条目，归档后学生端不再展示。
   - 小程序管理员页新增“知识库复核与发布”区域，可查看状态、发布、退回和归档。
   - 知识库录入表单新增检索词和敏感提示字段，便于提高搜索命中率并提示隐私事项。

2. 完善模板文件维护能力
   - 后端新增模板管理列表接口。
   - 支持模板元数据新增和编辑，包括模板名称、分类、文件类型、大小标识、下载链接和说明。
   - 支持归档模板文件。
   - 小程序管理员页新增“模板文件维护”区域，可新增、编辑、刷新和归档模板。
   - 未配置后端时，模板维护结果会写入本机缓存，便于演示和离线开发。

3. 完善党团流程配置能力
   - 后端新增流程节点管理列表接口。
   - 支持管理员维护入党、入团流程节点。
   - 支持维护节点编码、节点名称、短名称、说明、动作清单、提醒天数和排序。
   - 小程序管理员页新增“流程节点维护”区域，支持入党/入团切换、编辑已有节点和保存节点配置。
   - 未配置后端时，流程节点维护结果会写入本机缓存。

4. 完善学生党团进度维护能力
   - 后端新增学生党团进度管理查询接口。
   - 后端新增单个学生党团进度保存接口。
   - 支持按学号、姓名、班级搜索进度记录。
   - 支持维护当前节点、进入时间、下一节点时间、负责人和已完成动作。
   - 小程序管理员页新增“学生进度维护”区域，可编辑和保存学生个人进度。
   - 保存进度时同步更新学生表中的 `party_stage` 或 `league_stage`。

5. 完善学生档案和隐私展示
   - 后端新增管理员学生档案查询接口。
   - 根据权限返回敏感数据明文或脱敏值。
   - 手机号、身份证号、生源地、户籍、导师/负责人、特殊学籍状态、备注等按权限控制。
   - 小程序管理员页新增“学生档案与隐私”区域，支持搜索学生并查看脱敏状态。
   - 对具备 `read_sensitive` 或 `manage_all` 权限的账号展示“敏感可见”，否则展示“已脱敏”。

6. 完善角色权限展示
   - 个人中心新增“敏感数据”可见状态。
   - 个人中心新增“当前权限”标签区。
   - 权限标签包括公开政策查询、个人党团进度、知识库与模板维护、老师复核发布、流程与进度维护、敏感档案查看、操作日志查看等。
   - 管理员入口继续按权限控制显示。

7. 优化小程序离线演示能力
   - 未配置后端时，知识库草稿继续使用本机缓存。
   - 未配置后端时，模板文件维护、流程节点维护和学生进度维护均支持本机缓存。
   - 统一服务层异步返回方式，避免页面在 API 模式和离线模式下表现不一致。
   - 修复网络异常兜底路径中的递归风险。

8. 后端接口和服务扩展
   - 更新 `backend/src/routes/knowledgeRoutes.js`，增加知识库管理、编辑、退回和归档接口。
   - 更新 `backend/src/routes/templateRoutes.js`，增加模板管理、新增/编辑和归档接口。
   - 更新 `backend/src/routes/processRoutes.js`，增加流程节点管理和学生进度维护接口。
   - 更新 `backend/src/routes/profileRoutes.js`，增加学生档案管理查询接口。
   - 更新对应 service 层，补充参数校验、状态映射、权限相关数据处理和脱敏逻辑。

9. 小程序管理员页面扩展
   - 更新 `miniprogram/pages/admin/` 页面逻辑、结构和样式。
   - 管理员维护页从原先的导入和草稿演示，扩展为可操作的 P0 管理工作台。
   - 页面包括维护概览、上传策略、批量导入、知识库录入、知识库复核、模板维护、流程节点维护、学生进度维护、学生档案与隐私、操作日志和后续接口。
