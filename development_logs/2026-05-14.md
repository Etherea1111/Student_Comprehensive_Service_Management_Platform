# 2026-05-14 开发日志

## 今日目标

继续完善微信小程序前端用户界面，并新增适合小程序正式部署的登录系统。

## 完成内容

1. 前端 UI 优化
   - 统一全局背景、卡片、按钮、标签和 section 标题样式。
   - 优化首页为工作台式布局，增加实名状态、主操作按钮和核心模块卡片。
   - 优化问答页搜索面板、标准答复卡、模板文件卡和移动端布局。
   - 优化党团流程页，增加进度条、阶段计数、提醒卡和时间线状态。
   - 优化理论自测页，增加作答进度卡、选项状态和底部提交区域。
   - 优化个人中心和管理员维护页，提升入口、表单、日志和草稿展示层级。

2. 新增登录系统
   - 将登录页设为小程序首屏。
   - 默认提供微信登录。
   - 提供学号密码登录，账号固定为 10 位数字学号。
   - 新增首次微信登录后的学号和姓名绑定流程。
   - 新增修改密码页面。
   - 新增找回密码页面。
   - 未配置后端时保留“演示模式进入”按钮，便于开发者工具本地查看。

3. 后端认证能力扩展
   - 新增学号密码登录接口。
   - 新增修改密码接口。
   - 新增找回密码申请和重置密码确认接口。
   - 新增 PBKDF2 密码哈希工具。
   - 微信绑定接口改为使用 Bearer token 中的 openid，不信任前端传入 openid。
   - 学生导入时自动生成学号账号和初始密码。
   - 导入已有学生时不覆盖已修改过的密码。

4. 数据库更新
   - `users` 表新增密码哈希、是否必须改密、密码更新时间、最后登录时间等字段。
   - `users.openid` 改为可为空，以支持先导入学号账号、后绑定微信。
   - 新增 `password_reset_requests` 表，用于记录找回密码申请和重置凭证。

5. 文档更新
   - 更新 `Guidebook.md`，补充微信登录、学号密码登录、首次绑定、修改密码和找回密码说明。
   - 更新 `docs/backend-deployment.md`，补充认证接口、初始密码和找回密码生产环境注意事项。
   - 更新 `docs/miniprogram-development.md`，说明配置后端后的登录能力。

## 验证结果

- 所有后端和小程序 `.js` 文件通过 `node --check`。
- 所有小程序和后端 `.json` 文件通过 JSON 解析检查。
- 检查确认小程序 WXML 未保留复杂三元表达式或不稳定 Grid 布局。
