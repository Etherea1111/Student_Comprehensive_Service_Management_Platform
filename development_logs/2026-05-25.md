# 2026-05-25 开发日志

## 今日目标

根据新的小程序登录修改意见，移除微信登录渠道，改为账号注册、账号密码登录和首次学生身份绑定流程；同时关闭找回密码功能，并处理多个账号绑定同一学生信息时的账号合并规则。

## 完成内容

1. 重构小程序登录入口
   - 登录页从“微信登录 / 学号登录”调整为“账号登录 / 注册账号”。
   - 删除微信一键登录入口、微信登录说明和相关前端调用。
   - 新增注册账号表单，支持账号、昵称、密码、确认密码输入。
   - 注册成功后自动跳转至学生身份绑定页面。
   - 已注册账号可使用账号和密码登录。
   - 未绑定学生信息的账号登录后会自动进入绑定页面。

2. 调整学生身份绑定流程
   - 绑定页文案从“首次微信登录绑定”调整为“首次注册账号绑定”。
   - 绑定时仍校验 10 位学号和学生姓名。
   - 绑定成功后清除本地待绑定标记并进入首页。
   - 如当前登录账号仍处于待绑定状态，访问业务页时会被统一拦截并跳转至绑定页。

3. 实现同一学生多账号处理规则
   - 后端绑定逻辑按账号注册时间和账号 ID 判断较早账号。
   - 当多个账号绑定至同一学生信息时，保留注册时间较早的账号。
   - 注册时间较新的账号优先物理删除。
   - 如较新账号已有外键历史记录导致无法删除，则禁用该账号、释放学生绑定和账号名唯一占用。
   - 绑定接口返回较早账号的登录态，前端自动切换至较早账号继续登录。

4. 新增账号注册认证能力
   - 后端新增 `/api/auth/register` 接口。
   - 后端账号密码登录接口改为使用 `accountName`，兼容旧调用中的 `studentNo` 字段。
   - 新增账号名规范校验：4-32 位字母、数字或下划线。
   - 注册和登录接口统一签发 JWT，JWT 中写入账号名、学生信息、角色和权限。
   - 未绑定普通学生账号的权限被收窄为 `read_public`，避免绕过绑定访问个人业务能力。

5. 调整数据库结构
   - `users` 表新增 `account_name` 账号字段。
   - 移除旧 `openid` 字段和对应索引。
   - 移除 `password_reset_requests` 找回密码表。
   - 新增 `lower(account_name)` 唯一索引，避免账号大小写重复。
   - 初始化脚本会将已有学生账号的 `account_name` 回填为学号，无法匹配学生的旧账号回填为 `user_账号ID`。
   - 学生批量导入时继续生成默认账号和初始密码，并在账号名为空时回填学号。

6. 暂停找回密码功能
   - 删除小程序 `forgot-password` 页面文件。
   - 从 `app.json` 移除找回密码页面路由。

   - 登录页提示当前暂不提供找回密码功能。

7. 清理微信登录相关能力

## 补充：内置独立超级管理员账号

## 补充目标

为项目初始化、演示和后续联调提供一个不依赖学生信息绑定的超级管理员账号。该账号应具备当前项目支持的全部权限，但不允许在小程序内修改密码。

## 补充完成内容

1. 新增超级管理员种子账号
   - 在 `backend/db/seed.sql` 中新增独立超级管理员账号。
   - 登录账号为 `2024000001`。
   - 初始密码为 `123456`，以 PBKDF2 哈希形式保存到种子数据中。
   - 账号显示名为“超级管理员”。
   - 账号角色为 `super_admin`，通过现有权限解析逻辑获得当前系统全部权限。
   - 账号 `student_id` 固定为空，不绑定 `students` 表中的学生信息。
   - 种子脚本支持重复执行：若账号已存在，则更新为不绑定学生、超级管理员角色、固定密码和禁用改密状态；若不存在，则插入账号。

2. 新增禁止改密字段
   - 在 `users` 表新增 `password_change_disabled boolean not null default false` 字段。
   - 通过 `alter table ... add column if not exists` 保证旧数据库可平滑补字段。
   - 内置超级管理员账号将该字段设置为 `true`。

3. 调整认证和绑定逻辑
   - 登录接口返回 `passwordChangeDisabled` 信息。
   - 登录接口将 `bindingRequired` 限定为“普通学生且未绑定学生信息”，避免独立管理员账号被错误要求绑定学生。
   - 绑定学生接口对非学生账号直接返回无需绑定，不再允许超级管理员账号绑定学生档案。
   - JWT 中写入 `passwordChangeDisabled`，认证中间件解析该标记。

4. 禁止超级管理员账号修改密码
   - 后端 `changePassword` 接口查询 `password_change_disabled`。
   - 当账号禁用改密时，接口直接拒绝修改密码请求。
   - 小程序个人资料接口返回 `canChangePassword`。
   - “我的”页根据 `canChangePassword` 隐藏“修改密码”入口。
   - 若用户直接进入修改密码页，页面会提示“该账号不支持修改密码”并返回首页。

5. 兼容无学生信息的管理员资料展示
   - 个人资料接口在没有学生绑定时，使用 `account_name` 作为展示用 `studentNo`。
