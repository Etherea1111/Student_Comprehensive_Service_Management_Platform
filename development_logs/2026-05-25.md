# 2026-05-25 开发日志

## 今日目标

根据新的小程序登录修改意见，移除微信登录渠道，改为账号注册、账号密码登录和首次学生身份绑定流程；同时关闭找回密码功能，并处理多个账号绑定同一学生信息时的账号合并规则。

## 完成内容

1. 重构小程序登录入口
   - 登录页从“微信登录 / 学号登录”调整为“账号登录 / 注册账号”。
   - 删除微信一键登录入口、微信登录说明和相关前端调用。
   - 新增注册账号表单，支持账号、昵称、密码、确认密码输入。
   - 注册成功后自动跳转至学生身份绑定页面。
   - 已注册账号可使用账号和密码登录。
   - 未绑定学生信息的账号登录后会自动进入绑定页面。

2. 调整学生身份绑定流程
   - 绑定页文案从“首次微信登录绑定”调整为“首次注册账号绑定”。
   - 绑定时仍校验 10 位学号和学生姓名。
   - 绑定成功后清除本地待绑定标记并进入首页。
   - 如当前登录账号仍处于待绑定状态，访问业务页时会被统一拦截并跳转至绑定页。

3. 实现同一学生多账号处理规则
   - 后端绑定逻辑按账号注册时间和账号 ID 判断较早账号。
   - 当多个账号绑定至同一学生信息时，保留注册时间较早的账号。
   - 注册时间较新的账号优先物理删除。
   - 如较新账号已有外键历史记录导致无法删除，则禁用该账号、释放学生绑定和账号名唯一占用。
   - 绑定接口返回较早账号的登录态，前端自动切换至较早账号继续登录。

4. 新增账号注册认证能力
   - 后端新增 `/api/auth/register` 接口。
   - 后端账号密码登录接口改为使用 `accountName`，兼容旧调用中的 `studentNo` 字段。
   - 新增账号名规范校验：4-32 位字母、数字或下划线。
   - 注册和登录接口统一签发 JWT，JWT 中写入账号名、学生信息、角色和权限。
   - 未绑定普通学生账号的权限被收窄为 `read_public`，避免绕过绑定访问个人业务能力。

5. 调整数据库结构
   - `users` 表新增 `account_name` 账号字段。
   - 移除旧 `openid` 字段和对应索引。
   - 移除 `password_reset_requests` 找回密码表。
   - 新增 `lower(account_name)` 唯一索引，避免账号大小写重复。
   - 初始化脚本会将已有学生账号的 `account_name` 回填为学号，无法匹配学生的旧账号回填为 `user_账号ID`。
   - 学生批量导入时继续生成默认账号和初始密码，并在账号名为空时回填学号。

6. 暂停找回密码功能
   - 删除小程序 `forgot-password` 页面文件。
   - 从 `app.json` 移除找回密码页面路由。
   - 删除前端找回密码入口和服务方法。
   - 删除后端找回密码申请、确认重置接口和 service 方法。
   - 登录页提示当前暂不提供找回密码功能。

7. 清理微信登录相关能力
   - 删除后端 `/api/auth/wechat-login` 路由。
   - 删除后端微信 code 换 openid 逻辑。
   - 删除后端环境配置中的 `WECHAT_APP_ID` 和 `WECHAT_APP_SECRET` 读取。
   - 更新 `.env.example`，不再要求配置微信登录密钥。
   - 演示数据中的 `openid` 改为 `accountName`。

8. 更新相关查询和权限逻辑
   - 个人资料接口返回 `accountName`。
   - 管理员学生档案查询改为选择同一学生较早的活跃账号，避免多账号造成重复学生记录。
   - 个人党团进度查询增加用户未禁用条件。
   - 认证中间件增加 `accountName` 解析，并对未绑定账号做权限收窄。

9. 更新项目文档
   - 更新 `Guidebook.md`，将登录说明改为账号注册、密码登录和首次学生信息绑定。
   - 更新 `docs/backend-deployment.md`，同步认证接口、环境变量和生产注意事项。
   - 更新 `docs/miniprogram-development.md`，同步小程序登录能力说明。
   - 明确当前版本暂不提供找回密码功能。

## 补充：P1 通知聚合与精准推送基础建设

## 补充目标

在 P0 认证、知识库、模板、党团流程和导入能力基本完成后，开始进入 P1 阶段，优先建设“信息集成与精准推送”模块的后端基础能力，为后续小程序通知中心和管理员通知维护页提供数据结构与接口支撑。

## 补充完成内容

1. 新增通知模块数据库结构
   - 新增 `announcement_tags` 表，用于维护通知标签，例如就业、实习、评奖评优、学院事务等。
   - 新增 `announcements` 表，用于保存通知标题、摘要、正文、来源、优先级、发布状态、发布时间和过期时间。
   - 新增 `announcement_tag_relations` 表，用于支持一条通知关联多个标签。
   - 新增 `announcement_targets` 表，用于记录通知目标范围，支持全体、角色、年级、专业、班级、培养层次和指定学号。
   - 新增 `announcement_reads` 表，用于记录学生已读状态，支持后续显示未读消息。
   - 新增 `announcement_deliveries` 表，用于预留小程序、邮件、短信等多渠道投递状态记录。
   - 为通知发布查询、过期时间、目标匹配和投递记录补充索引，提升后续列表和筛选查询效率。

2. 新增通知后端服务层
   - 新增 `backend/src/services/announcementService.js`，集中封装通知列表、详情、已读、标签、管理列表、创建/更新、发布、撤回和归档逻辑。
   - 学生端通知查询只返回已发布、已到发布时间、未过期且符合目标范围的通知。
   - 管理端通知查询支持按状态和关键词筛选，并返回标签、目标范围和阅读数量。
   - 创建或更新通知时支持同步维护标签和目标范围，避免前端分别调用多个接口造成数据不一致。
   - 标签维护支持新增和更新，后续可接入管理员维护界面。

3. 新增通知后端路由
   - 新增 `backend/src/routes/announcementRoutes.js`。
   - 学生端接口包括通知列表、通知标签、通知详情和标记已读。
   - 管理端接口包括通知列表、通知创建/更新、标签维护、发布、撤回和归档。
   - 管理端接口复用现有 `manage_public_content` 权限，保证班团骨干、辅导员和超级管理员可以维护公开通知内容。
   - 对创建、发布、撤回、归档、标签维护和已读动作接入现有操作日志中间件。

4. 接入 Express 应用
   - 在 `backend/src/app.js` 中挂载 `/api/announcements` 路由。
   - 保持与现有 `/api/knowledge`、`/api/templates`、`/api/processes` 等模块一致的路由组织方式。
