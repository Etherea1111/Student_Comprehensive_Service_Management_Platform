# 学院学生综合服务与党团管理小程序使用手册

## 1. 小程序打开方式

本项目当前包含微信小程序前端和后端服务骨架，可通过微信开发者工具运行小程序，并可在部署后端后接入真实数据库与接口。

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择本仓库根目录。
4. AppID 可使用测试号，也可替换为真实小程序 AppID。
5. 点击“编译”后即可在模拟器中使用。

项目源码位于 `miniprogram/` 目录，工程配置文件为 `project.config.json`。

## 2. 首页使用方法

进入小程序后，首页会展示当前用户的学院、专业、年级和角色信息。

首页主要入口包括：

- 智能问答与政策知识库：进入政策问答、常用模板下载页面。
- 党团事务流程管理：进入入党、入团流程和个人进度页面。
- 今日提醒：查看当前党团阶段的下一节点提醒。
- 后续模块预留：展示后续可继续开发的信息推送、电子证明审批、学业预警等接口方向。

## 3. 智能问答与政策知识库

点击底部导航栏“问答”进入该模块。

### 3.1 查询政策问题

1. 在搜索框中输入关键词，例如“奖学金”“入党”“查档”“宿舍”。
2. 点击“搜索”。
3. 页面会返回匹配的标准答复。
4. 可通过上方分类按钮筛选不同类别的问题。

每条答复会展示：

- 问题标题
- 政策分类
- 标准答复
- 标签
- 维护单位
- 更新时间
- 官方链接复制按钮

如问题涉及休学、户籍、手机号、身份证号等敏感信息，页面会给出提示，建议联系负责老师处理，不在公开问答中填写隐私内容。

### 3.2 使用模板文件

在“常用模板”区域可以查看证明材料、活动预算表、活动简报等模板。

使用方式：

1. 选择模板分类。
2. 找到需要的模板。
3. 点击“下载/打开”。
4. 若示例链接无法访问，小程序会自动复制模板链接，便于后续替换为真实文件地址。

## 4. 党团事务流程管理

点击底部导航栏“党团”进入该模块。

### 4.1 查看入党/入团流程

页面顶部可切换：

- 入党流程
- 入团流程

切换后，页面会展示对应流程的阶段说明。

入党流程示例包括：

- 入党申请人
- 入党积极分子
- 发展对象
- 预备党员
- 正式党员

入团流程示例包括：

- 入团申请人
- 团员发展对象
- 共青团员

### 4.2 查看个人进度

页面会展示当前用户所在阶段，包括：

- 当前阶段
- 进入时间
- 下一节点时间
- 负责人
- 阶段进度
- 已完成动作
- 待完成动作

已完成的动作会用不同颜色标识，便于学生确认材料和学习任务是否遗漏。

### 4.3 查看关键节点提醒

“关键节点提醒”区域会提示当前阶段接下来需要关注的时间点和材料要求。

示例：

- 积极分子阶段需要按期提交思想汇报。
- 团员需要完成智慧团建信息维护和年度教育评议。

提醒仅用于辅助管理，正式要求仍以学院和老师通知为准。

### 4.4 理论自测

点击“开始自测”进入理论自测页面。

使用方式：

1. 逐题选择答案。
2. 完成所有题目后点击“提交答案”。
3. 页面会显示得分、正确/需复习状态和答案解析。
4. 可点击“重新作答”再次练习。

当前题库为演示数据，后续可由管理员导入官方题库。

## 5. 我的页面

点击底部导航栏“我的”进入个人中心。

该页面展示：

- 姓名
- 学号
- 班级
- 学院
- 专业
- 年级
- 当前角色
- 政治面貌
- 管理员账号绑定状态

页面还提供快捷入口：

- 政策问答与模板
- 我的党团流程
- 管理员维护

普通学生默认只能查看个人相关内容；具备权限的班团骨干、老师或超级管理员可以进入管理员维护页面。

## 6. 管理员维护页面

具备管理权限的账号可从“我的”页面进入“管理员维护”。

当前管理员页面提供以下功能：

### 6.1 维护概览

展示当前系统中的演示数据概览，例如：

- 知识库条目数量
- 模板文件数量
- 流程配置数量
- 待复核内容数量

### 6.2 上传策略

展示当前文件上传规则：

- 支持 `doc`、`docx`、`xls`、`xlsx`、`pdf`
- 单个文件上限为 30MB
- 上传内容需要复核
- 遵循“谁上传，谁维护”的原则

### 6.3 知识库录入演示

管理员可录入知识库草稿：

1. 填写问题标题。
2. 填写分类。
3. 填写标签。
4. 填写标准答复。
5. 填写官方链接。
6. 可选择 Word、Excel 或 PDF 文件。
7. 点击“提交待复核草稿”。

提交后，草稿会显示在“本机草稿”区域。

注意：配置后端地址后，草稿会提交到服务器知识库草稿接口；未配置后端地址时，草稿会临时保存在本机小程序缓存中，便于离线开发。

### 6.4 操作日志

管理员页面会展示演示操作日志，例如：

- 谁更新了知识库条目
- 谁导入了模板
- 谁调整了流程提醒

后续接入后台后，应由服务器统一记录管理员操作日志。

## 7. 后台操作说明

本节说明后续接入真实后台管理系统后的常用操作。当前小程序前端已预留角色、权限、导入和题库服务接口，正式上线时应由后台系统完成数据导入、权限配置和日志记录。

### 7.1 后台账号和权限分配

后台账号应采用“一人一号”，管理员账号不应多人共用。团支书等班团骨干的管理权限建议绑定到后台管理员账号，而不是仅依赖个人微信昵称。

推荐角色和权限如下：

| 角色 | 适用对象 | 主要权限 |
| --- | --- | --- |
| 普通学生 | 本学院学生 | 查看公开知识库、模板、个人党团进度、参加理论自测 |
| 班团骨干 | 团支书、班长、学生组织骨干 | 维护非保密知识库内容、上传公开材料、查看本人负责范围内的操作记录 |
| 班主任/辅导员 | 辅导员、班主任 | 查看学生必要信息、复核知识库内容、维护流程进度、处理学生事务 |
| 学院领导 | 学院相关领导 | 查看统计数据、查看必要敏感信息、查看操作日志 |
| 超级管理员 | 系统维护人员、指定老师 | 维护学生信息、角色权限、模板、标签、知识库、流程配置和系统日志 |

权限配置建议按以下步骤操作：

1. 进入后台“账号与权限”页面。
2. 通过学号、工号、手机号或微信 OpenID 查询账号。
3. 绑定真实姓名、身份类型、学院、班级或负责范围。
4. 选择角色，例如“普通学生”“班团骨干”“辅导员”“超级管理员”。
5. 按需勾选细分权限，例如“知识库维护”“模板维护”“流程配置”“敏感信息查看”“操作日志查看”。
6. 保存后由系统记录操作日志，包括操作人、被修改账号、修改前角色、修改后角色、修改时间。

建议细分权限字段：

| 权限编码 | 权限说明 |
| --- | --- |
| `read_public` | 查看公开信息 |
| `read_own_progress` | 查看本人党团进度 |
| `quiz` | 参加理论自测 |
| `manage_public_content` | 维护公开知识库和模板 |
| `audit_content` | 复核知识库和上传材料 |
| `manage_process` | 配置党团流程和节点提醒 |
| `import_students` | 导入学生基础信息 |
| `import_quiz` | 导入理论自测题库 |
| `read_sensitive` | 查看敏感学生信息 |
| `view_statistics` | 查看统计数据 |
| `view_operation_records` | 查看操作日志 |
| `manage_all` | 超级管理员权限 |

敏感权限如 `read_sensitive`、`manage_all` 应只分配给确有工作需要的老师或超级管理员。

### 7.2 学生信息导入

学生信息建议通过 Excel 批量导入。后台应提供“下载模板”“上传文件”“预览校验”“确认导入”四个步骤。

操作步骤：

1. 在后台进入“学生信息管理”。
2. 点击“下载导入模板”。
3. 按模板填写学生信息。
4. 上传 Excel 文件。
5. 后台自动校验必填字段、学号唯一性、手机号格式、身份证号格式和枚举值。
6. 管理员在预览页面确认新增、更新、异常数据。
7. 点击“确认导入”。
8. 系统记录导入日志，包括导入人、文件名、成功条数、失败条数、导入时间。

学生信息 Excel 格式规范：

| 字段名 | 是否必填 | 示例 | 说明 |
| --- | --- | --- | --- |
| `student_no` | 是 | `2024001001` | 学号，系统内唯一 |
| `name` | 是 | `李同学` | 学生姓名 |
| `college` | 是 | `信息学院` | 当前项目固定为信息学院 |
| `major` | 是 | `计算机科学与技术` | 专业名称 |
| `class_name` | 是 | `计科2401` | 行政班级 |
| `grade` | 是 | `2024级` | 年级 |
| `education_level` | 否 | `本科` | 本科、硕士、博士等 |
| `political_status` | 否 | `共青团员` | 群众、共青团员、中共预备党员、中共党员等 |
| `party_stage` | 否 | `activist` | 入党阶段编码 |
| `league_stage` | 否 | `member` | 入团阶段编码 |
| `phone` | 否 | `13800000000` | 敏感字段，需加密存储 |
| `id_card` | 否 | `110101200001010000` | 敏感字段，需加密存储 |
| `birthplace` | 否 | `北京市` | 敏感字段 |
| `household_register` | 否 | `北京市海淀区` | 敏感字段 |
| `ethnicity` | 否 | `汉族` | 可按权限展示 |
| `advisor` | 否 | `王老师` | 导师或负责老师 |
| `student_status` | 是 | `在读` | 在读、休学、延毕、毕业、转学、退学 |
| `is_alumni` | 否 | `否` | 是否校友 |
| `awards` | 否 | `校级三好学生` | 可选获奖情况 |
| `remark` | 否 | `无` | 备注 |

党团阶段编码建议：

| 字段 | 编码 | 含义 |
| --- | --- | --- |
| `party_stage` | `none` | 暂无入党流程 |
| `party_stage` | `applicant` | 入党申请人 |
| `party_stage` | `activist` | 入党积极分子 |
| `party_stage` | `development` | 发展对象 |
| `party_stage` | `probationary` | 预备党员 |
| `party_stage` | `full` | 正式党员 |
| `league_stage` | `none` | 暂无入团流程 |
| `league_stage` | `applicant` | 入团申请人 |
| `league_stage` | `candidate` | 团员发展对象 |
| `league_stage` | `member` | 共青团员 |

导入规则：

- `student_no` 不能为空，且必须唯一。
- `student_status` 不能为空。
- 已存在学号时，后台应提示是“更新已有学生”还是“跳过已有学生”。
- 手机号、身份证号、生源地、户籍、休学、延毕、毕业、转学、退学等信息应按敏感数据处理。
- 毕业、转学、退学学生的信息可以留存，但默认不在学生端公开展示。
- 导入失败的数据应生成错误报告，标明行号、字段名和失败原因。

### 7.3 理论自测题库导入

理论自测题库建议支持 Excel 批量导入，后续也可支持从题库系统导出后导入。

操作步骤：

1. 在后台进入“理论自测题库”。
2. 点击“下载题库导入模板”。
3. 按模板填写题目。
4. 上传 Excel 文件。
5. 后台校验题干、选项、正确答案和解析。
6. 管理员预览题目。
7. 确认导入后，题目进入题库。
8. 如题库需要正式发布，应由老师或具备复核权限的管理员点击“发布”。

题库 Excel 格式规范：

| 字段名 | 是否必填 | 示例 | 说明 |
| --- | --- | --- | --- |
| `question_id` | 否 | `party_001` | 题目编号；为空时系统自动生成 |
| `category` | 是 | `党建基础` | 题目分类 |
| `difficulty` | 否 | `easy` | `easy`、`medium`、`hard` |
| `stem` | 是 | `入党积极分子通常需要提交哪类材料？` | 题干 |
| `option_a` | 是 | `思想汇报` | A 选项 |
| `option_b` | 是 | `课程退选表` | B 选项 |
| `option_c` | 否 | `宿舍维修单` | C 选项 |
| `option_d` | 否 | `校园卡挂失单` | D 选项 |
| `answer` | 是 | `A` | 正确答案，单选题填写 A/B/C/D |
| `explanation` | 否 | `思想汇报是培养考察中的常见材料。` | 答案解析 |
| `status` | 否 | `draft` | `draft` 草稿，`published` 已发布 |
| `source` | 否 | `学院党委` | 题目来源 |
| `updated_at` | 否 | `2026-05-13` | 更新时间 |

题库导入规则：

- `stem`、`option_a`、`option_b`、`answer` 不能为空。
- `answer` 必须在已有选项范围内。
- 单选题至少需要 2 个选项，最多建议 4 个选项。
- 题干重复时，后台应提示“跳过”“覆盖”或“作为新题导入”。
- 正式发布前建议由党团学老师复核，避免题目或答案不准确。
- 已发布题目如需修改，应保留修改记录。

### 7.4 知识库和模板导入

知识库可通过后台表单逐条录入，也可通过 Excel 批量导入。政策原文、通知附件、模板文件可通过 Word、Excel、PDF 上传。

知识库 Excel 格式规范：

| 字段名 | 是否必填 | 示例 | 说明 |
| --- | --- | --- | --- |
| `title` | 是 | `奖学金评定常见问题` | 问题标题 |
| `category` | 是 | `奖助学金` | 分类 |
| `tags` | 否 | `奖学金,评奖评优` | 多个标签用逗号分隔 |
| `keywords` | 否 | `奖学金,综测,成绩排名` | 搜索关键词 |
| `answer` | 是 | `以学院当年通知为准。` | 标准答复 |
| `official_link` | 否 | `https://example.edu.cn/notice` | 官方链接 |
| `owner` | 是 | `学院团委` | 维护单位 |
| `sensitive_hint` | 否 | `涉及个人情况请联系辅导员。` | 敏感问题提示 |
| `status` | 否 | `draft` | `draft` 草稿，`published` 已发布 |
| `updated_at` | 否 | `2026-05-13` | 更新时间 |

模板文件元数据格式规范：

| 字段名 | 是否必填 | 示例 | 说明 |
| --- | --- | --- | --- |
| `template_name` | 是 | `在读证明申请模板` | 模板名称 |
| `category` | 是 | `证明材料` | 模板分类 |
| `file_type` | 是 | `docx` | 文件类型 |
| `file_url` | 是 | `https://example.edu.cn/templates/proof.docx` | 文件下载地址 |
| `description` | 否 | `适用于普通在读证明申请。` | 模板说明 |
| `owner` | 是 | `学生工作办公室` | 维护单位 |
| `updated_at` | 否 | `2026-05-13` | 更新时间 |

上传文件规则：

- 支持 `doc`、`docx`、`xls`、`xlsx`、`pdf`。
- 单个文件建议不超过 30MB。
- 谁上传，谁维护。
- 涉密文件不建议直接上传，可用文字说明代替附件。
- 发布前应由具备复核权限的老师确认内容。

### 7.5 后台操作日志要求

以下操作必须记录日志：

- 新增、修改、删除管理员账号。
- 分配或撤销权限。
- 导入学生信息。
- 修改学生敏感信息。
- 导入或发布理论自测题库。
- 新增、修改、发布知识库条目。
- 上传、替换、删除模板文件。
- 修改党团流程配置和提醒时间。

日志字段建议：

| 字段名 | 说明 |
| --- | --- |
| `log_id` | 日志编号 |
| `operator_id` | 操作人账号 |
| `operator_name` | 操作人姓名 |
| `operator_role` | 操作人角色 |
| `action` | 操作类型 |
| `target_type` | 操作对象类型 |
| `target_id` | 操作对象编号 |
| `before_value` | 修改前内容，可按需脱敏 |
| `after_value` | 修改后内容，可按需脱敏 |
| `created_at` | 操作时间 |
| `ip_address` | 操作 IP |
| `device_info` | 设备信息 |

敏感信息写入日志时应脱敏，例如身份证号只保留前 6 位和后 4 位。

## 8. 隐私和权限说明

小程序按照需求预留了角色权限设计。

已预留角色包括：

- 普通学生
- 班团骨干
- 班主任/辅导员
- 学院领导
- 超级管理员

以下信息属于敏感信息，不应在学生端公开展示：

- 身份证号
- 手机号
- 生源地
- 户籍
- 休学状态
- 延毕状态
- 退学、转学、毕业后的保密信息

老师和管理员查看敏感数据时，应按权限控制，并保留操作日志。

## 9. 当前工程状态与上线准备

当前版本已从单纯前端原型推进为可继续落地的小程序工程，已包含：

- 原生微信小程序前端。
- Node.js 后端 API 骨架。
- Kingbase/PostgreSQL 兼容数据库建表脚本。
- 微信登录和学生实名绑定接口。
- 角色权限校验。
- 学生信息导入接口。
- 理论自测题库导入接口。
- 知识库草稿、复核发布接口。
- 党团流程配置和个人进度接口。
- 操作日志接口。
- 导入模板文件。

正式上线前还需要完成以下准备：

1. 申请或确认真实微信小程序 AppID。
2. 部署后端服务，并配置 HTTPS 域名。
3. 在微信公众平台配置 request 合法域名。
4. 配置 `miniprogram/config/env.js` 中的 `apiBaseUrl`。
5. 准备 Kingbase 数据库实例，执行 `backend/db/schema.sql` 和必要初始化数据。
6. 配置后端 `.env` 中的 `DATABASE_URL`、`JWT_SECRET`、`DATA_CRYPTO_KEY`、`WECHAT_APP_ID`、`WECHAT_APP_SECRET`。
7. 将模板文件链接替换为学校或学院可访问的真实文件地址。
8. 由老师审核后导入正式知识库、学生信息和理论题库。
9. 确认敏感字段加密、日志留存、管理员权限分配和数据备份策略。

## 10. 后续开发建议

后续继续开发时，建议优先完成：

1. 将微信登录中的本地 mock 兜底替换为正式 `jscode2session` 调用和异常处理。
2. 增加后台 Web 管理端页面，方便老师在 PC 端维护学生、题库、模板和知识库。
3. 接入受控文件服务或对象存储，用于模板、政策文件和附件管理。
4. 完善学生党团进度批量导入和批量更新功能。
5. 增加知识库全文检索或向量检索能力，但回答仍应优先引用标准答案和官方文件。
6. 完成电子证明审批、精准通知和学业预警等后续模块。
7. 建立正式的数据备份、日志审计和权限复核制度。

前端页面已经通过 `services/` 层隔离数据来源。配置真实后端地址后，小程序会优先调用后端 API；未配置后端时仍保留本地数据兜底，便于开发和排查问题。
