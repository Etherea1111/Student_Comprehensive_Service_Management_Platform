## 1. 总体说明

- 所有表使用 **UTF-8** 编码，排序规则为 `zh_CN.UTF-8`。
- 主键统一使用自增整型 `SERIAL`（KingbaseES 兼容 PostgreSQL 语法）。
- 时间字段统一使用 `TIMESTAMPTZ`（带时区），存储 UTC 时间。
- 学生敏感字段（身份证号、手机号）在应用层 AES-256 加密后存储，数据库中保存密文。
- 软删除：重要业务表增加 `deleted_at TIMESTAMPTZ DEFAULT NULL`，`NULL` 表示未删除。

## 4. 索引设计

```sql
-- 用户表
CREATE UNIQUE INDEX idx_users_openid ON users(openid);
CREATE UNIQUE INDEX idx_users_student_id ON users(student_id) WHERE student_id IS NOT NULL;
CREATE INDEX idx_users_grade_major ON users(grade, major);

-- 知识库
CREATE INDEX idx_knowledge_category ON knowledge_articles(category);
CREATE INDEX idx_knowledge_keywords ON knowledge_articles USING GIN(keywords);

-- 通知
CREATE INDEX idx_notices_pushed_at ON notices(pushed_at DESC);
CREATE INDEX idx_notices_target_grade ON notices USING GIN(target_grade);
CREATE INDEX idx_notice_reads_user ON notice_reads(user_id);

-- 党团
CREATE UNIQUE INDEX idx_party_user ON party_applications(user_id);
CREATE INDEX idx_party_stage ON party_applications(current_stage);
CREATE INDEX idx_reminders_unsent ON reminders(user_id) WHERE is_sent = FALSE;

-- 证明审批
CREATE INDEX idx_certificates_user_status ON certificates(user_id, status);

-- 学业
CREATE INDEX idx_academic_records_user ON academic_records(user_id);
CREATE INDEX idx_academic_warnings_user ON academic_warnings(user_id) WHERE is_resolved = FALSE;
CREATE INDEX idx_courses_module_major ON courses(module_type, major);
```

## 5. 安全与加密约定

| 字段 | 处理方式 |
|------|----------|
| `users.id_card_enc` | 应用层 AES-256-GCM 加密，密钥由环境变量注入，不入库 |
| `users.phone_enc` | 同上 |
| `certificates.filled_data` | JSONB 存储展示用快照，不含原始身份证号明文 |
| 日志文件 | Django logging 配置过滤敏感字段，禁止打印身份证号、手机号 |

## 6. 命名规范

| 规则 | 示例 |
|------|------|
| 表名：小写下划线，模块前缀 | `party_applications`、`academic_warnings` |
| 字段名：小写下划线 | `student_id`、`created_at` |
| 外键字段：`关联表单数_id` | `user_id`、`plan_id` |
| 加密字段：`字段名_enc` 结尾 | `id_card_enc`、`phone_enc` |
| 时间字段：统一 `TIMESTAMPTZ` | `created_at`、`deleted_at` |
| 枚举值：英文小写下划线字符串 | `full_member`、`pending` |
| 索引：`idx_表名_字段` | `idx_users_openid` |
