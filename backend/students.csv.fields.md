# students.csv 字段说明

`backend/students.csv` 用于学生信息批量导入。若 CSV 表头与后端导入接口不一致，以后端导入接口为准。

当前后端学生导入接口读取字段如下：

| 字段名 | 含义 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `student_no` | 学号 | 是 | 10 位数字学号，用于账号生成和学生唯一匹配。 |
| `name` | 姓名 | 是 | 与学号共同用于身份核验。 |
| `college` | 学院 | 否 | 为空时后端默认写入“信息学院”。 |
| `major` | 专业 | 是 | 例如“计算机科学与技术”。 |
| `class_name` | 班级 | 是 | 例如“计科2401”。 |
| `grade` | 年级 | 是 | 例如“2024级”。 |
| `education_level` | 培养层次 | 否 | 例如“本科”“硕士”“博士”。 |
| `political_status` | 政治面貌 | 否 | 例如“共青团员”“群众”“中共党员”。 |
| `party_stage` | 入党流程阶段编码 | 否 | 为空时后端默认 `none`；常用值如 `applicant`、`activist`、`development`、`probationary`、`full`。 |
| `league_stage` | 入团流程阶段编码 | 否 | 为空时后端默认 `none`；常用值如 `applicant`、`candidate`、`member`。 |
| `phone` | 手机号 | 否 | 敏感字段，导入后由后端加密保存。 |
| `id_card` | 身份证号 | 否 | 敏感字段，导入后由后端加密保存。 |
| `birthplace` | 生源地 | 否 | 敏感字段，导入后由后端加密保存；不要使用 `hometown` 表头。 |
| `household_register` | 户籍 | 否 | 敏感字段，导入后由后端加密保存。 |
| `ethnicity` | 民族 | 否 | 需求中属于可公开基础信息。 |
| `advisor` | 导师/负责人 | 否 | 按需求作为敏感或受限信息展示。 |
| `student_status` | 学籍状态 | 是 | 例如“在读”“休学”“毕业”“转学”“退学”。特殊状态展示时需要按权限控制。 |
| `is_alumni` | 是否校友/离校 | 否 | 可填写“是”“true”“TRUE”“1”表示是；其他值按否处理。 |
| `awards` | 获奖情况 | 否 | 学生可选扩展信息。 |
| `remark` | 备注 | 否 | 可能包含敏感说明，展示时需要按权限控制。 |

当前不被学生导入接口读取的字段：

- `gender`
- `birth_date`
- `hometown`

如后续确需导入这些字段，应先扩展数据库表结构、导入服务和权限展示逻辑，再更新 CSV 表头。
